# 经验总结

1. 以后使用PPT、Markdown和Demo并进的组合，PPT可用于讲解，列出差异、核心步骤等，简单明了，只需把握大体，具体细节写在Markdown里，与代码相照应，代码中的注释也与Markdown里相吻合。

2. 把握好广度与深度，过广会导致听众无法一下子接受那么多内容，过深会导致晦涩难懂（或许），应多多请教其他人，咨询意见，做好功课。

3. 调整心态，心性的磨炼不可停歇。
